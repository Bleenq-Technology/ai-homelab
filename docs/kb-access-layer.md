# Shared KB Access Layer

How any app or agent (Apollo, trading-engine, n8n flows, future services) queries the shared `kb_*`
knowledge bases — **one contract, one embedding standard, no per-app RAG**. This is the interim
realisation of [`kb-standards.md`](kb-standards.md) §5 (`list_kbs` / `search_kb`); the Qdrant MCP
(todos #4) will later expose the same two operations over MCP.

It has three pieces, all driven by the registry:

1. **Registry** — [`kb-manifest.json`](kb-manifest.json): the source of truth (name, description,
   collection, source repo, `enabled`). Read it to know what's searchable.
2. **`kb-list`** — an n8n webhook that returns the live registry.
3. **`kb-search`** — an n8n webhook that embeds a query with `bge-m3` and searches one collection.

> Embedding happens **server-side** with `bge-m3` (1024-dim, Cosine) — callers send plain text, so
> every reader is automatically §1-compliant. Do **not** embed client-side.

## Endpoints

Base URL:
- **From a container on the `ai` docker network** (preferred): `http://n8n:5678`
- **External / off-network:** `https://n8n.pdx.sanctioned.tech`

### `GET /webhook/kb-list`
Returns the manifest. Use entries where `enabled == true`; route on `name` + `description`, query by
`collection`.
```json
{ "embedder": "bge-m3", "dim": 1024, "distance": "Cosine",
  "kbs": [ { "name": "kb_homelab_docs", "enabled": true, "collection": "kb_homelab_docs",
             "description": "Homelab infrastructure docs — …" }, … ] }
```

### `POST /webhook/kb-search`
Body:
```json
{ "collection": "kb_homelab_docs", "query": "how is SSO enforced?", "topK": 6 }
```
(`topK` optional, default 6.) Response:
```json
{ "collection": "kb_homelab_docs", "query": "how is SSO enforced?",
  "results": [
    { "id": "…", "score": 0.49,
      "payload": {
        "content": "# Keycloak SSO\n…",
        "metadata": { "source": "core/keycloak/README.md",
                      "repository": "https://github.com/Bleenq-Technology/ai-homelab-infra",
                      "branch": "main", "loc": { "lines": { "from": 1, "to": 32 } } } } }
  ] }
```
`results` is ranked by cosine similarity (higher = closer). `payload.content` is the chunk text;
`payload.metadata.source` is the repo-relative file path (use it for citations).

## Usage pattern

1. `GET /webhook/kb-list` (or read `kb-manifest.json`) → choose the relevant `collection`(s) from the
   descriptions. For broad questions, search several and merge.
2. `POST /webhook/kb-search` per chosen collection with the user's text.
3. Cite `metadata.source` (+ `loc.lines`); fall back gracefully when `results` is empty.

### Examples
```bash
# discover
curl -s http://n8n:5678/webhook/kb-list | jq '.kbs[] | select(.enabled) | {name, collection}'
# search
curl -s -X POST http://n8n:5678/webhook/kb-search -H 'Content-Type: application/json' \
  -d '{"collection":"kb_trading_docs","query":"turtle trading position sizing","topK":5}'
```
```python
import requests
BASE = "http://n8n:5678"   # or https://n8n.pdx.sanctioned.tech off-network
def search_kb(collection, query, k=6):
    r = requests.post(f"{BASE}/webhook/kb-search",
                      json={"collection": collection, "query": query, "topK": k}, timeout=60)
    r.raise_for_status()
    return r.json()["results"]   # [{score, payload:{content, metadata}}]
def list_kbs():
    return [k for k in requests.get(f"{BASE}/webhook/kb-list", timeout=30).json()["kbs"] if k["enabled"]]
```

## Notes / boundaries

- **Auth:** the webhooks are currently **unauthenticated on the internal `ai` network** (Qdrant/LiteLLM
  keys stay server-side in n8n credentials — callers never see them). If exposed beyond the homelab,
  gate them (add a token check in the workflow + an `Authorization` header). The OpenWebUI chat pipe and
  the `kb-chat` agent both reach KBs this same way.
- **Embedding standard is enforced here**, not by callers — swapping the embedder is a one-place change.
- **App responsibilities** (kb-standards §9): apps *read* shared `kb_*` via this layer; they own their
  private `<app>_*` collections separately (still `bge-m3`/1024/Cosine).
- **Source of truth for the endpoints:** generated by `docker/ai/n8n/build_kb_workflows.py`
  (`kb-search`, `kb-list`); see `docker/ai/n8n/workflows/`.
